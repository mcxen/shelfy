# Orden 卡片流程设计

Orden 的可视化不再复刻 YAML 表单，而是把一条规则表达成一条可读的卡片轨道：

```text
文件来源  →  中间条件  →  执行动作
  1..n          0..n         1..n
```

## 交互模型

1. **小卡片负责识别，检查器负责编辑**
   - 卡片只显示类型、关键参数摘要和风险状态，保证多步骤时仍能快速扫读。
   - 选择卡片后，当前卡片的完整参数才在下方展开，避免所有 YAML 参数同时占据页面。

2. **来源也是流程节点**
   - 多个文件或目录来源按卡片处理，可用左右按钮切换。
   - 来源卡片同时展示目标类型、路径和是否包含子目录；点击卡片进入路径编辑。

3. **条件与动作支持动态卡片组**
   - 条件卡使用 `IF` 标记，并通过“全部 / 任一 / 排除”表达 AND / OR / NONE。
   - 动作卡显示执行序号，卡片顺序就是实际执行顺序。
   - 卡片间连线上的加号可在指定位置插入步骤；拖拽用于快速重排，上移/下移保留为键盘友好的精确操作。
   - 来源、条件、动作三段在宽屏显示明确方向连线，窄屏按阅读顺序自然纵向排列。

4. **快捷卡片库代替记忆指令名**
   - “添加条件 / 添加动作”先展示常用卡片，再展示完整能力。
   - 每个卡片类型包含人类可读名称、用途说明和可运行的默认参数。
   - 永久删除、Shell、废纸篓等会改变文件状态的卡片有显式风险标识。

5. **YAML 是高级能力，不是默认入口**
   - 新建配置、使用模版后默认进入 Visual 卡片模式。
   - Source 模式继续保留完整 Orden YAML 能力，用于可视模型尚未覆盖的复杂配置。

## 模版中心

模版中心以“想完成什么”作为第一层，而不是以 YAML 文件作为第一层：

- 按整理、自动化、备份、维护和自定义目标浏览。
- 模版卡直接展示 `来源 → 条件 → 动作` 摘要。
- 详情使用与编辑器一致的三段卡片预览，并显示“保留原件 / 会移动或改写 / 运行前请检查”的影响等级。
- 主操作为“使用并继续调整”，创建配置后直接进入 Visual 编辑器。
- YAML 编辑、另存和删除收进“高级模版选项”；从现有配置保存模版使用应用内 Dialog，不使用系统 prompt。

## 响应式原则

- 规则元数据在窄窗口折行；来源、条件、动作在宽窗口并排，在窄窗口纵向排列。
- 每组动态卡片自身横向滚动，不依赖页面横向滚动。
- 700px 下保留完整可编辑能力；900px 下操作按钮允许换行；约 1120px 后展示三段并排流程。
- 模版中心在窄窗口先展示所选模版详情，再展示模版列表；宽窗口使用主列表 + 右侧详情。
