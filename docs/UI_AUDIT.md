# Shelfy UI 修复审计

> 审计依据：2026-07-10 提供的 6 张界面截图、`Settings.tsx`、设置页子组件、基础 UI 组件和主题 token。截图仅作为会话依据，不复制进仓库。

## 范围与验收基线

- 保留当前顶部浮动导航，不恢复旧侧栏；`UI_SPEC.md` 中“固定侧栏”属于待同步的规范漂移。
- 所有项目在浅色/深色、中文/英文以及 `700×500`、`900×650`、`1280×800` 下验收。
- 交互控件必须有可见焦点；图标按钮必须有可访问名称，含义不明显时提供 Tooltip。
- 页面主按钮高度控制在 32px，紧凑工具栏按钮 28px；除表单输入外，按钮不以 `w-full` 或大段固定宽度占据工具栏。
- Card 使用 8px 圆角，普通控件使用 6–8px 圆角；不再通过多层 Card 和重复边框表达同一层级。

## P0 — 阻断与可访问性

### UI-P0-01 Rules / Orden 响应式溢出

- 截图依据：截图 1、4、5 的右侧操作列被压缩或接近裁切。
- 代码根因：规则名称单元格 `min-w-72`，操作列同时放置多个完整文字按钮；Orden 表格也将试跑、预览、更多操作并排。
- 修复：Rules 表格与 Orden 配置、任务、执行历史表格统一仅在 `>=900px` 显示；窄窗口使用保留预览、试跑、菜单、运行、编辑、删除、开关和日志展开操作的卡片行。编辑器命令与配置名/保存/删除区可换行。
- 验收：700px 宽时所有操作可见；900px 宽时表头、状态和操作列不裁切；不存在页面级横向滚动。

### UI-P0-02 Menu 被容器裁切

- 截图依据：截图 4 中 Orden “更多”菜单被表格和 Card 边界裁切。
- 代码根因：自制 `MenuPopup` 使用父容器内 `absolute right-0 top-full`，无 Portal、可用高度或碰撞翻转。
- 修复方向：基于 Base UI/coss Menu，统一通过 Portal 渲染并启用 Positioner 碰撞处理。
- 验收：菜单在表格末行、窗口右缘和底部打开时始终处于可视区域，必要时自动翻转并限制最大高度。

### UI-P0-03 Menu 键盘与语义

- 截图依据：截图 4 的菜单属于关键配置操作入口。
- 代码根因：旧菜单是普通 `div/button` 状态拼装，没有 menu 语义、方向键漫游、焦点返回和完整 Escape 行为。
- 修复方向：使用 coss Menu primitive；保留 Group、Label、Separator 和 destructive 语义。
- 验收：Enter/Space 打开，方向键移动，Escape 关闭，选择后焦点回到触发器；读屏可识别菜单及菜单项。

## P1 — 视觉系统与页面布局

### UI-P1-01 主题层级和品牌颜色

- 截图依据：6 张截图中背景、Card、边框均集中在相近深绿色；正文偏奶油黄，主按钮偏灰绿。
- 代码根因：暗色 token 的 background/card/popover/muted 色相和亮度过近，foreground 与 ring 使用偏黄颜色；模版卡片绕过 token 使用红紫橙渐变。
- 修复方向：暗色背景转为中性松针黑，Card/Popover 提升明度差，正文转中性灰白；主色统一为清晰鼠尾草绿；模版只使用主题 token 的语义色阶。
- 验收：背景、Card、浮层、边框四层可辨；正文无明显黄偏；代码中不存在模版红紫橙渐变字符串。

### UI-P1-02 按钮尺寸体系

- 截图依据：截图 1 的权限 CTA、截图 4/5 的新增和编辑按钮、截图 2 的操作按钮过宽过重。
- 代码根因：基础 Button 默认 36px 高和 16px 横向 padding，页面操作又重复使用完整文字；缺少 `xs`、`icon-sm` 等工具栏层级。
- 修复方向：默认 32px、`sm` 28px，增加 `xs/icon-sm`；表格行操作图标化并提供 Tooltip；权限 CTA 使用短文案。
- 验收：工具栏按钮不挤压主内容；权限 CTA 最大内容宽度且高度 28px；编辑/更多操作形成一致图标组。

### UI-P1-03 圆角、Card 和密度

- 截图依据：截图 2–4 中 Hero、导航、表单组和嵌套 Card 视觉体积过大。
- 代码根因：大量 `rounded-xl/2xl/3xl`、嵌套 Card、重复背景和边框；Table 单元格 padding 与空状态偏大。
- 修复方向：Card 统一 8px，表单分区优先使用 section/divider；表格头 36px，行垂直 padding 8–10px；空状态压缩。
- 验收：同一内容层级最多一层 Card 边框；900×650 首屏能展示主要列表和下一分区起始内容。

### UI-P1-04 Rules 页面

- 截图依据：截图 1、5。
- 代码根因：长功能描述重复展示文件夹范围与状态；行内有完整编辑按钮；监控文件夹使用二级 Card。
- 修复方向：主列只保留规则名与匹配摘要，范围独立截断；状态由开关和可访问名称表达；行操作图标化；文件夹项改紧凑行。
- 验收：900px 表格无横向滚动；单行高度明显低于原实现；700px 卡片模式完整可操作。

### UI-P1-05 General 页面

- 截图依据：截图 2。
- 代码根因：Select、Slider、数字输入默认撑满容器；开机启动只占半栏；定时器配置纵向堆叠且原生 number spinner 暴露。
- 修复方向：设置项采用 label/value 对齐网格并设置 320–560px 最大宽度；开机启动占满整行；数字输入统一隐藏 spinner；定时器改紧凑两列/工具栏布局。
- 验收：1280px 下控件不会无限拉长且无大片无意义空白；700px 自动单列；数字框无浏览器 spinner。

### UI-P1-06 Template Gallery

- 截图依据：截图 3。
- 代码根因：硬编码高饱和渐变；Hero 与 80px 图标过大；选中详情和同一模版卡片重复；YAML 默认完整展开；`role=button` 容器内嵌加号按钮；创建/删除使用原生 prompt/confirm。
- 修复方向：模版元数据保存语义 tone，前端映射主题 token；压缩 Hero；选中项从下方网格排除；YAML 折叠；整卡选择和加号作为并列按钮；创建使用内联命名区，删除使用可访问 Alert Dialog。
- 验收：中文页面没有未定义英文 badge；卡片无品牌外渐变；键盘可分别聚焦卡片与加号；textarea 有 label；不调用原生 prompt/confirm。

### UI-P1-07 Orden 页面

- 截图依据：截图 4。
- 代码根因：列表外层 Card 再包表格 Card；空任务表固定大块留白；状态区分依赖弱色 badge；三个行操作并排。
- 修复方向：每个区块单层 Card；列表操作图标化；状态增加文本/图标语义；空任务使用紧凑提示；700px 使用卡片列表。
- 验收：菜单无溢出；900×650 首屏能同时看到配置区和任务区标题；700px 所有操作可达。

### P1 页面密度进度（2026-07-11）

- 已完成：Rules、General、Template、Orden、History。五页均已收紧 Card、表格或记录行、工具栏和空状态；工具栏在窄窗口换行，General 控件在桌面保持合理最大宽度并在窄窗口铺满。
- Orden 的下一轮重点从密度切换为流程可操作性：来源、条件、动作增加方向连线，步骤支持指定位置插入和拖拽重排；上下移动按钮继续作为键盘可达的等价操作。
- 修复普通 Vite 预览无条件调用 Tauri IPC/通知而卡在 loading 的问题，恢复浏览器内 700px / 1280px 响应式回归能力；真实 Tauri 窗口仍需最终验收。

### UI-P1-08 History 页面

- 截图依据：截图 6。
- 代码根因：筛选、全部撤销和清空历史挤在一行右侧；记录 Card 三列固定；路径使用 `break-all`；engine/action 直接显示底层英文值。
- 修复方向：工具栏允许换行并将危险操作分组；记录行压缩；路径单行截断并保留 title；标准 engine/action 映射本地化文案。
- 验收：700px 工具栏不互相挤压；路径不逐字断行；中文界面不直接显示 `rules/orden/move/copy/ignore` 等底层枚举。

### UI-P1-09 Navigation / Permission

- 截图依据：截图 1、2、6。
- 代码根因：活动 Tab 仅使用接近背景色的浅层 surface；品牌、导航、关闭按钮使用 16px 以上视觉圆角；权限提示在两个 Tab 重复并使用长 CTA。
- 修复方向：活动 Tab 使用 primary token 的明确底色与前景；统一 8px 圆角；权限提示保持一行、短 CTA，并只在有权限错误或 Rules 入口展示说明。
- 验收：活动 Tab 在两种主题下一眼可辨；CTA 不超过自身文案宽度；导航在 700px 可横向访问且不挤压关闭按钮。

## P2 — 交互一致性与规范同步

### UI-P2-01 图标按钮和焦点

- 代码根因：部分删除、关闭、运行按钮只有 `aria-label`，但没有可视 Tooltip；不同 primitive 的 focus ring 不一致。
- 修复方向：关键图标按钮统一 Tooltip；Button/Input/Select/Menu 使用同一 ring token。
- 验收：鼠标悬停可理解操作；Tab 导航时焦点轮廓清楚且不被裁切。

### UI-P2-02 原生 prompt/confirm

- 代码根因：Rules、Template、Orden 备注、History 等流程直接调用 `window.prompt/confirm`。
- 修复方向：本轮先消除截图覆盖的 Template 创建/删除和列表危险操作原生弹窗，统一 Dialog/Alert Dialog；其余遗留项继续跟踪。
- 验收：审计覆盖页面的核心创建/删除流程有标题、说明、取消和确认按钮，并正确返回焦点。

### UI-P2-03 表单标签与本地化

- 代码根因：YAML textarea 只有视觉邻近文本或完全无标签；若干 Cron/Tags 和底层枚举直接显示英文。
- 修复方向：textarea 通过 `Label htmlFor` 关联；产品术语可保留，界面枚举必须走 i18n。
- 验收：表单控件有程序化名称；中英文切换后没有缺 key 或未定义枚举文案。

### UI-P2-04 规范漂移

- 代码根因：`docs/UI_SPEC.md` 仍描述固定侧栏，而当前产品已采用顶部浮动导航。
- 修复方向：本审计记录差异；本轮不重写规范，待界面稳定后单独同步。
- 验收：后续规范更新明确顶部导航、密度、按钮尺寸和 700px 响应式策略。
